Use the following guidelines and sample file to define the environment variables for starting {{prodname}} on the host. For more help, see the 
{%- if include.install == "container" %}
 [{{nodecontainer}} configuration reference](/reference/node/configuration).
{% else %}
 [Felix configuration reference](/reference/felix/configuration).
{% endif %}

For the Kubernetes datastore set the following:

| Variable | Configuration guidance |
|----------|------------------------|
| KUBECONFIG | Path to kubeconfig file to access the Kubernetes API Server |

{% if include.install == "container" %}
:::note

If using certificates and keys, you will need to volume mount them into the container at the location specified by the paths mentioned above.

:::
{% endif %}

Sample `EnvironmentFile` - save to `/kubernetes/calico/calico.env`

```shell
{{datastore_type}}=kubernetes
CALICO_NODENAME=""
NO_DEFAULT_POOLS="true"
CALICO_IP=""
CALICO_IP6=""
CALICO_AS=""
CALICO_NETWORKING_BACKEND=bird
```
