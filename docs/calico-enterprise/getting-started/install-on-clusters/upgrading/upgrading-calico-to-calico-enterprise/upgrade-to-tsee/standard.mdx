## Prerequisites

Ensure that your Kubernetes cluster is running with open source Calico on the latest {{ calico_minor_version | append: '.x' }} release.
If not, follow the [Calico upgrade documentation](https://docs.projectcalico.org/{{calico_minor_version_with_path}}/maintenance/kubernetes-upgrade) before continuing.

{{prodname}} only supports clusters with a Kubernetes datastore.
Please contact Tigera Support for assistance upgrading a cluster with an `etcdv3` datastore.

If your cluster already has {{prodname}} installed, follow the [Upgrading {{prodname}} from an earlier release guide](/maintenance/kubernetes-upgrade-tsee) instead.

For hybrid Linux and Windows clusters, ensure that your Windows nodes have at least 4 cores, 8GB RAM.

## Upgrade Calico to {{prodname}}

:::note

GKE upgrades from open source Calico are not currently supported.

:::

### Install the {{prodname}} license

In order to use {{prodname}}, you must install the license provided to you by Tigera.

```
kubectl create -f </path/to/license.yaml>
```

You can now monitor progress with the following command:

```
watch kubectl get tigerastatus
```
