# Binary install with package manager

## Big picture

Install {variables.prodname} on non-cluster hosts using a package manager.

## Value

Packaged binaries of {variables.prodname} are easy to consume and upgrade. This method automatically configures the init system to keep Felix running.

## Before you begin...

1. Ensure the {variables.prodname} datastore is up and accessible from the host
1. Ensure the host meets the minimum [system requirements](../requirements.mdx)
1. If your system is not an Ubuntu- or RedHat-derived system, you will need to choose a different install method.
1. If you want to install {variables.prodname} with networking (so that you can communicate with cluster workloads), you should choose the [container install method](container.mdx)
1. Install `kubectl` (for Kubernetes datastore) or [Install and configure `calicoctl`](../../../operations/calicoctl/index.mdx) for etcd3 datastore.

## How to

This guide covers installing Felix, the {variables.prodname} daemon that handles network policy.

### Step 1: Install binaries

_PPA requires_: Ubuntu 14.04 or 16.04

<CodeBlock language='bash'>
{`sudo add-apt-repository ppa:project-calico/{variables.ppa_repo_name}
sudo apt-get update
sudo apt-get upgrade
sudo apt-get install calico-felix`}
</CodeBlock>

_RPM requires_: RedHat 7-derived distribution

<CodeBlock language='bash'>
{`cat > /etc/yum.repos.d/calico.repo \<\<EOF
[calico]
name=Calico Repository
baseurl=http://binaries.projectcalico.org/rpm/{variables.ppa_repo_name}/
enabled=1
skip_if_unavailable=0
gpgcheck=1
gpgkey=http://binaries.projectcalico.org/rpm/{variables.ppa_repo_name}/key
priority=97
EOF

yum install calico-felix`}
</CodeBlock>

Until you initialize the database, Felix will make a regular log that it
is in state "wait-for-ready". The default location for the log file is
`/var/log/calico/felix.log`.

### Step 2: Configure the datastore connection

<EnvironmentFile
  nodecontainer={variables.nodecontainer}
  target='felix'
/>

Modify the included init system unit to include the `EnvironmentFile`. For example, on systemd, add the following line to the `[Service]` section of the `calico-felix` unit.

```bash
EnvironmentFile=/etc/calico/calico.env
```
