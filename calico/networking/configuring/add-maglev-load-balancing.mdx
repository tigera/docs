# Add Maglev load balancing to a service

This guide shows you how to enable Maglev-style load balancing to a service.
Maglev's consistent-hashing load balancing algorithm minimizes the amount of dropped packets and improves perfomance.
This is especially useful for maintaining sessions during a load balancer failover.

## Prerequisites

* Your Kubernetes cluster uses $[prodname] for networking.
* All your nodes are running on Linux
* $[prodname] is configured to use the eBPF data plane.
* You've enabled [direct server return mode](https://docs.tigera.io/calico/latest/operations/ebpf/enabling-ebpf#try-out-direct-server-return-mode).
* You have the name and namespace of a Service of type `LoadBalancer`.
* You have `kubectl` access to your cluster.

## Enable consistent-hash load balancing for a service

To enable consistent-hash load balancing for a service, annotate the service by running the following command:

```bash
kubectl -n <namespace> annotate service <service> <placeholder-annotation>
```

Replace the following:
* `<namespace>`: The namespace of the service. (omit the `-n` flag for the `default` namespace).
* `<service>`: The name of the service.
