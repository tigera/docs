# calicoctl datastore migrate lock

This sections describes the `calicoctl datastore migrate lock` command.

Read the [calicoctl Overview](../../overview.mdx)
for a full list of calicoctl commands.

## Display the help text for 'calicoctl datastore migrate lock' command

Run `calicoctl datastore migrate lock --help` to display the following help menu for the
command.

```
Lock the datastore to prevent changes during migration

Usage:
  calicoctl datastore migrate lock [flags]

Flags:
  -c, --config string   Path to the file containing connection configuration in YAML or JSON format. (default "/etc/calico/calicoctl.cfg")
  -h, --help            help for lock

Global Flags:
      --allow-version-mismatch   Allow client and cluster versions mismatch
      --context string           The name of the kubeconfig context to use
  -l, --log-level string         Set the log level (panic, fatal, error, warn, info, debug) (default "panic")
```

### Examples

Lock the datastore to prepare it for migration so that any changes to the
data will not affect the cluster during migration.

```bash
calicoctl datastore migrate lock
```

### General options

```
-c, --config string   Path to the file containing connection configuration in YAML or JSON format. (default "/etc/calico/calicoctl.cfg")
```
