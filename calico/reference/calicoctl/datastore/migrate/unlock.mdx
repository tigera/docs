# calicoctl datastore migrate unlock

This sections describes the `calicoctl datastore migrate unlock` command.

Read the [calicoctl Overview](../../overview.mdx)
for a full list of calicoctl commands.

## Display the help text for 'calicoctl datastore migrate unlock' command

Run `calicoctl datastore migrate unlock --help` to display the following help menu for the
command.

```
Unlock the datastore to allow changes after migration

Usage:
  calicoctl datastore migrate unlock [flags]

Flags:
  -c, --config string   Path to the file containing connection configuration in YAML or JSON format. (default "/etc/calico/calicoctl.cfg")
  -h, --help            help for unlock

Global Flags:
      --allow-version-mismatch   Allow client and cluster versions mismatch
      --context string           The name of the kubeconfig context to use
  -l, --log-level string         Set the log level (panic, fatal, error, warn, info, debug) (default "panic")
```

### Examples

Unlock the datastore after migration to allow the Calico resources to affect
the cluster.

```bash
calicoctl datastore migrate unlock
```

### General options

```
-c, --config string   Path to the file containing connection configuration in YAML or JSON format. (default "/etc/calico/calicoctl.cfg")
```
