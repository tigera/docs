This section describes the `calicoctl cluster health-check` command.

Read the [calicoctl Overview]({{ site.baseurl }}/reference/calicoctl/overview)
for a full list of calicoctl commands.

Note, this command can be run from any location that has access to the cluster
(e.g. anywhere with kubeconfig).

## Displaying the help text for 'calicoctl cluster health-check' command

Run `calicoctl cluster health-check --help` to display the following help menu for the
command.

```
Usage:
  calicoctl cluster health-check [--since=<SINCE>] [--config=<CONFIG>]

Options:
  -h --help               Show this screen.
     --since=<SINCE>      Only collect logs newer than provided relative duration,
                          in seconds (s), minutes (m) or hours (h)
  -c --config=<CONFIG>    Path to the file containing connection configuration in
                          YAML or JSON format.
                          [default: /etc/calico/calicoctl.cfg]

Description:
  The cluster health-check command verify Calico Enterprise installation and configuration
  for the given cluster.
```
{: .no-select-button}

### Examples

```bash
calicoctl cluster health-check
```

An example response follows.

```
==== Begin health check. ====

```
