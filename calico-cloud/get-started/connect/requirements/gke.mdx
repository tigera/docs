### Verify system requirements

Although {{site.prodname}} checks that your cluster meets [System requirements]({{sitebase.url}}/get-started/requirements/system-requirements), verifying now can save you time troubleshooting.  

### Verify GKE settings

1. Verify that intranode visability is `Enabled`/true.

    ```bash
    gcloud container clusters describe <cluster-name> --flatten networkConfig.enableIntraNodeVisibility
    ```

1. Verify network policy is `Disabled`/true.
    
    ```bash
    gcloud container clusters describe <cluster-name> --flatten addonsConfig.networkPolicyConfig.disabled
    ```

1. Verify Dataplane V2 is `null`/not configured.

    ```bash
    gcloud container clusters describe <cluster-name> --flatten networkConfig.datapathProvider
    ```

### Verify IAM permissions
